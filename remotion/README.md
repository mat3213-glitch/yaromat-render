# TSX/Remotion движок — техника движения для пула разнообразия

Программная motion-графика (React/Remotion) как **новый медиум** в пуле: параллакс по глубине,
частицы, кинетическая типографика, генеративная графика — то, что ffmpeg-композ не даёт.
Рендер **только на GH Actions** (`tsx_sandbox.yml`), никогда на буке.

## Архитектура

```
remotion/
  src/
    contract.ts            ← КОНТРАКТ: props, что получает любой шаблон (граница песочницы)
    Root.tsx               ← реестр TEMPLATES (mimo добавляет 1 строку на шаблон)
    index.ts               ← registerRoot
    templates/
      KineticCard.tsx       ← референс-шаблон (образец)
      <NewTemplate>.tsx     ← сюда mimo пишет новые шаблоны
../tsx_templates.json      ← реестр ОДОБРЕННЫХ шаблонов (боевой пул)
../tsx_proposals.json      ← кандидаты на ревью (агент пишет сюда)
../tsx_scout_merge.py      ← ГЕЙТ: после OK yaromat вливает кандидата в реестр
```

## Поток (зеркало Style Scout)

1. **Поиск/разработка** — агент находит готовый TSX-эффект (GitHub topic `remotion`,
   awesome-remotion, react-bits…) ЛИБО генерит из фото (Pinterest/Openverse → palette/depth → TSX).
2. **Адаптация под контракт** — шаблон принимает РОВНО `TemplateProps` (`contract.ts`), бренд строчными, без неона.
3. **Тест в песочнице** — `tsx_sandbox.yml` (workflow_dispatch) рендерит на тест-арте → превью в TG-тред.
4. **Предложение** — запись в `tsx_proposals.json` (`{name, composition, source, note, preview, format}`).
5. **Гейт** — yaromat смотрит превью → OK → `tsx_scout_merge.py --names <...>` → шаблон в боевом реестре → push.

Шаблон НЕ попадает в пул, пока (а) его .tsx нет в `templates/` + строки в `Root.tsx`, и (б) merge не одобрен.

## Контракт шаблона (`contract.ts`)

Каждый шаблон — **чистая функция `props → кадры`**: без сети, без файловой системы, без секретов.
Получает: `artUrl` (staticFile), `trackTitle`, `brand`='yaromat', `palette` (приглушённая), `seed`, `format`.

Правила вайба (незыблемо): бренд строчными латиницей; НИКАКОГО неона; вайб = внутренняя глубина,
не одиночество; без масок/лиц/эзотерики.

## ЗАДАЧА СУБАГЕНТА `mimo` (второстепенная, песочница)

`mimo` (пишет на уровне Sonnet 4.6) — **автор TSX-шаблонов в песочнице**. Его работа:

1. По выданной мной спеке (тип эффекта + контракт) — написать `templates/<Name>.tsx`.
2. Добавить РОВНО одну строку в `Root.tsx` (`TEMPLATES`): import + объект `{id, component}`.
3. Прогнать `npm run lint` (tsc --noEmit) — компиляция без ошибок.
4. Дописать кандидата в `tsx_proposals.json`.
5. Доложить — что сделал; дальше песочница-рендер (GH) + ревью + гейт делаю я/yaromat.

### Почему mimo НЕ МОЖЕТ сломать проект (структурные гарантии, не «обещание в промпте»)

| Барьер | Как обеспечен |
|---|---|
| **Скоуп-тюрьма** | mimo пишет ТОЛЬКО в `remotion/src/templates/` + одну строку в `Root.tsx`. Рабочая папка ограничена `remotion/`. |
| **Нет боевого кода** | publish_gh, vzrosly_clip_job, styles.json, очередь, crontab, publish-репо — ВНЕ его песочницы. |
| **Нет секретов** | окружение mimo без токенов (TG/ЯД/VK/YT/GH-push). Физически не может ни опубликовать, ни запушить, ни писать на ЯД. |
| **Вывод инертен** | шаблон ничего не делает в проде, пока `tsx_scout_merge.py` (это запускаю я/yaromat) не вольёт его в реестр. mimo merge не запускает. |
| **Изолированный тест** | валидация в `tsx_sandbox.yml` — отдельный workflow; кривой шаблон валит свой рендер и не трогает боевые джобы/пул. |
| **Read-only на прод** | mimo может ЧИТАТЬ боевой код (выучить контракт/бренд), но писать — только в песочницу. |
| **Не на буке** | рендер на GH, не локально (бук Atom/1.8GB защищён). |
| **Запрет деструктива** | в его рабочем дереве нет prod-данных; `rm`/git push/деплой вне песочницы недоступны. |

Даже если mimo проигнорирует инструкцию — **отсутствие секретов + скоуп-тюрьма + ручной merge-гейт**
делают боевой пайплайн физически недосягаемым. Худший случай — кривой .tsx, который не пройдёт lint/sandbox-рендер.
